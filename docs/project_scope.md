# Project Scope

## Objective

The objective of this project was to improve inventory transparency, data consistency, and reporting quality for a small pharmacy inventory management scenario.

The project focuses on identifying operational data quality issues and supporting inventory monitoring through structured reporting workflows.

---

## Business Problems

The following issues were identified within the inventory dataset:

* Duplicate medicine records
* Missing supplier information
* Inconsistent category naming
* Invalid expiration date formats
* Low stock inventory risks
* Inactive and expired products
* Missing storage condition information

---

## Project Goals

* Improve inventory visibility
* Support low stock monitoring
* Increase operational reporting quality
* Detect data inconsistencies
* Create business-oriented dashboards
* Demonstrate structured data quality workflows

---

## Technologies Used

* Excel
* SQL
* Python (Pandas)
* Power BI

---

## Expected Outcome

The project demonstrates how simple data quality validation and reporting workflows can support more reliable operational decision-making in pharmacy-related inventory environments.
